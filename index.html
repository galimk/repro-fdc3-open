FDC3 Open Repro — SUPPORT-2343

Reproduces: addContextListener fails when "Allow Duplication" is enabled

Pages

/app-a.html — Caller app (invokes fdc3.open)
/app-b.html — Listener app (registers fdc3.addContextListener)

Repro Steps

  1. Add the apps below to your cloud-platform app directory (or inline config)
  2. Launch the platform with openfin-direct locally
  3. Open App B — verify it shows "FDC3 Ready — Listening"
  4. Open App A — click the fdc3.open() button
  5. Expected: App B shows the received context
  6. With allowDuplication ON + App B already open: Context is never received

App Directory Entries

Add these to your platform's app directory config (or paste into the apps endpoint response):

{
  "apps": [
    {
      "appId": "app-a",
      "title": "App A (FDC3 Open Caller)",
      "manifestType": "inline-view",
      "manifest": {
        "url": "http://localhost:8082/app-a.html",
        "fdc3InteropApi": "2.0"
      },
      "icons": [],
      "publisher": "repro"
    },
    {
      "appId": "app-b",
      "title": "App B (Context Listener)",
      "manifestType": "inline-view",
      "manifest": {
        "url": "http://localhost:8082/app-b.html",
        "fdc3InteropApi": "2.0"
      },
      "customConfig": {
        "allowDuplication": true
      },
      "interop": {
        "intents": {
          "listensFor": {}
        },
        "userChannels": { "broadcasts": [], "listensFor": ["fdc3.contact"] }
      },
      "icons": [],
      "publisher": "repro"
    }
  ]
}

What to Compare

Working case: Set "allowDuplication": false (or remove the field) in App B's config. Open App B, then call fdc3.open from App A → context arrives.

Broken case: Keep "allowDuplication": true. Open App B first, then call fdc3.open from App A → context never arrives (or AppTimeout error in App A).
